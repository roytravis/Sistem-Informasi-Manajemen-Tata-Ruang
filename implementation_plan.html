Implementation Plan

Migrate SIMANTRA from React SPA to Laravel Livewire 3
Date: 10 Maret 2026  |  Project: Sistem Informasi Manajemen Tata Ruang

1. Overview

Replace the separate React frontend (frontend-baru/) with Livewire 3 components inside the existing Laravel 12 backend. This eliminates all Node.js/npm dependencies — no build step required.

AspectCurrent (React SPA)Target (Livewire)
FrontendReact 19 + Vite 7 + TailwindLaravel Blade + Livewire 3 + Tailwind CDN
Build Toolnpm, Vite (requires Node.js)None — zero build step
AuthSanctum token (localStorage)Laravel session auth
API CommunicationREST API via AxiosDirect Eloquent (Livewire)
DeploymentStatic files + API serverSingle Laravel app
✅ API Routes Preserved The existing API routes (api.php) are NOT modified. They remain available for any mobile app or external integration.
⚠️ Auth Change The web frontend switches from Sanctum token auth to Laravel session auth. The API routes continue using Sanctum tokens. Both coexist.

2. Architecture

Current Architecture

[Browser] → React SPA (Vite) → axios HTTP → Laravel API (Sanctum tokens) → MySQL

Target Architecture

[Browser] → Laravel Blade + Livewire → Direct Eloquent → MySQL

All 15 React pages will be converted to Livewire full-page components. The cPanel hosting supports PHP 8.2+ and MariaDB 10.11 — fully compatible.

4. Phase 1 — Setup & Foundation

MODIFY composer.json
Add livewire/livewire: ^3.0 via composer require
NEW resources/views/layouts/app.blade.php
Main Blade layout: Tailwind CSS via CDN, @livewireStyles / @livewireScripts, navigation bar with role-based menus, notification dropdown, logout button.
NEW resources/views/auth/login.blade.php
Login page matching React LoginPage.jsx design. Session-based auth with NIP + password.
NEW app/Http/Controllers/Auth/LoginController.php
Handles login POST (session auth) and logout. Replaces Sanctum token flow for web.
MODIFY routes/web.php
Add all web routes: login/logout, plus Livewire full-page component routes with auth middleware.

5. Phase 2 — Core Livewire Components

NEW app/Livewire/PemegangManager.php
Replaces PemegangPage.jsx — CRUD with modal for add/edit, delete confirmation, search. Fields: nama_pelaku_usaha, nomor_identitas, kegiatan, alamat, desa/kelurahan, kecamatan, email, nomor_handphone.
NEW resources/views/livewire/pemegang-manager.blade.php
Blade template with table + modal form for Pemegang CRUD.
NEW app/Livewire/TimManager.php
Replaces TimPenilaiPage.jsx — Tim CRUD + member management (add/remove users with jabatan: Ketua Tim, Petugas Lapangan, Koordinator Lapangan).
NEW resources/views/livewire/tim-manager.blade.php
Card grid layout with create/edit modal and manage members modal.
NEW app/Livewire/PenilaianDashboard.php
Replaces PenilaianPage.jsx — Dashboard with status filters (pending/all), pagination, assign team modal, complex status badges with signature tracking, role-based action buttons.
NEW resources/views/livewire/penilaian-dashboard.blade.php
Dashboard table with tab filters and action buttons per role.

6. Phase 3-4 — Detail & Complex Form Pages

After Phase 2 is verified working, the remaining 10 pages will be migrated:

  • Phase 3: AddPenilaian, EditPenilaian, PenilaianDetail, EditApproval
  • Phase 4: BeritaAcaraPemeriksaan, AddBeritaAcara, BeritaAcaraPreview, FormulirAnalisis, BaHasilPenilaianInput, BaHasilPenilaianPreview

These are complex form pages with signatures, multi-step workflows, and print/preview functionality.

7. Verification Plan

Browser Testing (After Each Phase)

  • Open http://127.0.0.1:8000/login, enter credentials, verify redirect to dashboard
  • Navigate to /pemegangs — test add/edit/delete a pemegang
  • Navigate to /tims — test creating a tim and managing members
  • Navigate to /penilaian — verify dashboard with filter tabs and role-based buttons
  • Verify logout redirects to login page

cPanel Deployment

  • Upload Laravel project to cPanel via File Manager or Git
  • Set PHP version to 8.2+ via MultiPHP Manager
  • Run composer install via Terminal
  • Configure .env for MariaDB 10.11 database
  • Point document root to public/
